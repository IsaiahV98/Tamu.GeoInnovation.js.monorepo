The following web services provide access to the Texas A&M Geoservices geocoding system. The versions below are the simplified APIs and offer reduced input parameters and output values than do the advanced Geocoding APIs which provide additional metadata about the internal working of the Texas A&M Geoservices Geocoder as well as advanced options for requesting specific computational options, reference data sources, output hierarchies, etc. (i.e., all the science under the hood).

Most people probably don't need or care about the internal workings of geocoding systems, but they are provided in the advanced APIs in hopes that more people start to! If you are brave, here to view the advanced versions of the API's.

Quick Start

The Geoservices Geocoding API is available to developers in several ways:

  • Raw HTTP, REST, or SOAP requests
  • Client JavaScript API's
    • CDN (hosted)
    • NPM

Raw requests offer the greatest flexibility for developers allowing custom implementations based on their programming language of choice and consuming application formats (xml, csv, etc.)

Using the hosted version is best for applications that do not utilize a package manager like NPM. Simply add the script requirement to the head of your webpage and you're off to the races!

For projects utilizing the NPM package manager, module installation is done through its CLI and imported into your project. This has the benefit of including the API client library into your project and releasing network external dependencies that could impact your application performance and load times. Furthermore, the client API libraries are written with TypeScript so developers get the added benefit of JSDoc or native TypeScript typings in supported IDE's or Code Editors.

allowTies

Since API Version: 4.0

Flag to indicate to return ties as valid results or handle using the tieBreakingStrategy parameter. Ties are allowed by default. if false is provided, a valid strategy for tieBreakingStrategy must also be provided.

Example URL:

GET
{{ url }}?apiKey={{ apiKey }}&version={{ apiVersion }}&allowTies=true&streetAddress=9355 Burton Way&city=Beverly Hills&state=ca&zip=90210&format=json
filter_none play_circle_outline
link

tieBreakingStrategy

Since API Version: 4.0

The name of the tie-breaking strategy to use when ties are found. Supported values:

  • flipACoin
    • Choose and return one of the ties at random.
  • revertToHierarchy
    • Fail on a tie and match to the next level of the geographic hierarchy - parcel, then street, then ZIP, then city, etc.

Example URL:

GET
{{ url }}?apiKey={{ apiKey }}&version={{ apiVersion }}&tieBreakingStrategy=flipACoin&allowTies=false&streetAddress=9355 Burton Way&city=Beverly Hills&state=ca&zip=90210&format=json
filter_none play_circle_outline

censusYear

Since API Version: 2.96

Pipe-separated list of supported census data years to intersect with. Available years include:

  • 1990
  • 2000
  • 2010
  • allAvailable

The value allAvailable is a shorthand for all of the supported years.

Example URL:

GET
{{ url }}?apiKey={{ apiKey }}&version={{ apiVersion }}&censusYear=1990|2000&census=true&streetAddress=9355 Burton Way&city=Beverly Hills&state=ca&zip=90210&format=json
